## Summary\n\nThe workflows in `.github/workflows/` likely reference GitHub Actions using floating version tags (e.g. `actions/checkout@v4`). Pinning to specific commit SHAs is a supply-chain security best practice.\n\n## Affected Workflows\n\n- `.github/workflows/R-CMD-check.yaml`\n- `.github/workflows/pkgdown.yaml`\n- `.github/workflows/test-coverage.yaml`\n- `.github/workflows/copilot-setup-steps.yml`\n\n## Proposed Work\n\n- Replace all floating version tags with their corresponding full commit SHAs (e.g. `actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683`).\n- Consider using a tool like [Dependabot](https://docs.github.com/en/code-security/dependabot/working-with-dependabot/keeping-your-actions-up-to-date-with-dependabot) to automate future Action version bumps.\n\n## Why\n\nFloating tags can be silently updated to point at different (potentially malicious) commits. Pinning SHAs ensures reproducibility and security.
